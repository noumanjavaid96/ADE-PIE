# Annotated Confidence Score Logic: From Vision to Feasible Implementation

## Purpose of this Document

This document serves as the definitive guide for the **Confidence Score** feature. Its purpose is to create a single source of truth that bridges the client's visionary user journey with a technically sound and achievable implementation plan.

We deeply respect the original vision and have worked to preserve its spirit while ensuring the underlying logic is robust, measurable, and scalable. This annotated comparison is designed to provide full transparency into the necessary adjustments and the reasoning behind them.

---

## Updated Scoring Formula

The Confidence Score is calculated using a weighted average that gives more importance to recent events.

**Formula:** `New Score = (0.7 * Previous Score) + (0.3 * New Raw Value)`

-   **Previous Score:** The user's Confidence Score before the latest event.
-   **New Raw Value:** A value from -1.0 to 1.0 assigned to each user action (e.g., `quiz_answered` correct = `0.8`, `help_requested` = `-0.9`).
-   This formula ensures that the score is dynamic and responsive, reflecting the user's current state of understanding while still retaining a memory of their overall journey.
